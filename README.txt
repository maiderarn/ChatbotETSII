Dentro del zip ANEXO_III_Codigo se encuentra el archivo JavaScript con nombre "index" que será el programa principal del archivo. Además, habrá que tener abierta las 
carpetas "Models" y "node_modules". Como se explica en la memoria, el programa de node.js será el encargado de recibir el nombre de la Intent que
DialogFlow ha relacionado con la petición del usuario así como la información en modo de parámetros en caso de que haga falta consultar en la base
de datos de MongoDB. Para ello, habría que conectar el programa de JavaScript a Dialogflow mediante Ngrok, que permite crear una URL de tunelización que 
ha de ser conectada con DialogFlow especificandole a ésta última concretamente cuál es la URL. Para ello, se ha de disponer del usuario de DialogFlow.
Es por eso, que si el profesor desea realizar alguna prueba, debe contactar con el grupo para que conecte el proyecto de DialogFlow, o que en su defecto
le haga una demostración presencial del funcionamiento del programa.
